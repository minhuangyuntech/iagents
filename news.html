OpenClaw v2026.7.1 仍是今天最值得先讀的官方版本，因為它把 guided setup、連線檢查與 approval boundary 放進同一版，提醒學生「第一次能不能用」其實是 onboarding、憑證、模型與 fallback 一起成立。

2026-07-14 · OpenClaw 官方 Commit

OpenClaw 修正 OpenAI OAuth models 在 cloud workers 失敗的問題，這是一個很好的 worker-auth 教材：同一個模型路由若沒有把授權 profile 投影到正確執行環境，雲端 worker 就會在真正推理前先壞掉。

帶學生看這則 commit 時，可多問三件事：auth route 是在哪一層決定、worker 端看到的是哪個 profile、修補是把路由投影清楚還是只在表面重試。

2026-07-14 · Hermes Agent 官方 Commit

Hermes 新增 auxiliary 各任務可獨立設定 reasoning_effort，這很適合拿來教學生：vision、web_extract、compression 不一定要共用同一個 thinking budget，任務成本與品質可以分欄設計。

這則 commit 不只加設定鍵，也把文件與測試一起補上；課堂上可直接比較「全域預設」與「每個 task 自訂」對 latency、token 與結果品質的影響。

2026-07-14 · Hermes Agent 官方 Commit

Hermes 修正 MCP ResourceLink 要顯示真實 wire tool name，並在 error path 把 resource 文字內容露出來；這很適合教學生：如果錯誤只剩一句泛用訊息，agent 根本無法知道下一步該讀哪個資源。

這則修改把「可呼叫的真實工具名稱」與「錯誤中的資源內容」都保留下來，正好能拿來比較 observability 做得夠不夠，會不會把 agent 引到不存在的 tool。

2026-07-13 · OpenClaw Official Release

OpenClaw v2026.7.1 is still the best official item to study first today because it packages guided setup, connection checks, and approval boundaries into one release-level lesson: first-run success depends on onboarding, credentials, model routing, and fallback all working together.

In class, turn the release notes into a first-run checklist: which step validates the provider, which step persists config, and which step must preserve earlier choices after an interrupted setup.

2026-07-14 · OpenClaw Official Commit

OpenClaw fixes OpenAI OAuth models failing on cloud workers, which is a compact worker-auth lesson: if the auth profile is not projected into the real execution environment, the worker breaks before inference even starts.

A useful prompt is to separate three layers: where the auth route is decided, which profile the worker actually sees, and whether the repair improves projection of the route or merely hides the failure behind retries.

2026-07-14 · Hermes Agent Official Commit

Hermes now lets each auxiliary task set its own reasoning_effort, making it a strong teaching example that vision, web_extract, and compression do not need to share the same thinking budget or latency target.

The change also updates tests and docs, so students can compare a global default versus per-task tuning and ask when higher reasoning is worth extra time and tokens.

Source (Hermes Agent Official Commit) →
2026-07-14 · Hermes Agent Official Commit

Hermes fixes MCP ResourceLink rendering so the marker points to the real wire tool name and still surfaces resource text on the error path, a useful lesson that observability fails when an agent is shown a fake tool name or an empty error shell.

This is a clean way to teach why the next action must remain grounded in real callable tool names and why error payloads should preserve the resource text that explains what to fetch or inspect next.

2026-07-14 · External Report (The Verge)

The Verge reports that Grok Build was uploading users’ entire codebases to a cloud bucket, making it today’s clearest outside comparison point: coding agents are not only about generation quality, but also about whether source code, paths, and artifacts are allowed to leave the local machine at all.

For student discussion, add three fixed columns: what data was uploaded, who could access it, and whether the user was informed beforehand. That turns “powerful tool” into a real boundary-and-audit question.
