學習目標

Ollama 共用原則

  • 所有使用者共用 localhost:11434。
  • 模型只下載一次。
  • 所有學員共用同一份模型檔。
  • 不必為每位學員重複安裝模型。

安全性建議

學員不可 sudo

避免誤改系統、降低環境失控風險。

Docker 僅教師可用

避免容器資源競爭與權限擴張。

只管理自己的 Home

降低互相覆蓋設定、刪檔與環境污染。

Node / Python 建議

  • Node：使用 nvm 或 mise，不要直接綁系統 Node。
  • Python：使用 uv + venv，避免所有人共用套件。

Golden Image 與 /etc/skel

可將 aliases、Git、uv、nvm、VS Code CLI、Ollama Client、OpenClaw Template、Claude Code Template、範例專案放在 /etc/skel，讓每位新學員帳號建立時都擁有相同起始環境。

實作練習

請學員描述：若你要管理 20 位學員的帳號，哪些設定該由教師預先處理？哪些設定應保留給學員自己完成？
